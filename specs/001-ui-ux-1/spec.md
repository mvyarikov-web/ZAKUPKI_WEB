# Спецификация фичи: UI/UX для «архивов как папок» в веб‑интерфейсе

Feature Branch: 001-ui-ux-1  
Created: 2025-09-30  
Status: Draft  
Input: «Сделать поведение архивов (ZIP/RAR) в UI таким же, как у папок, с предпросмотром и индикаторами: (1) архивы кликабельны как папки, (2) заголовки/контекст содержат имя архива и путь внутри, (3) предпросмотр со сниппетами и подсветкой совпадений, (4) “светофор” наличия совпадений для папок/файлов, (5) под именем показывать char_count, (6) кнопка “очистить все” сбрасывает поиск и подсветки, (7) показывать путь папки, из которой загружали». 

Примечание локализации: все тексты и сообщения интерфейса — на русском языке.

## Сценарии и тестирование

### Основная пользовательская история
Как пользователь, я хочу видеть и открывать архивы (ZIP/RAR) в дереве файлов так же, как обычные папки, просматривать найденные совпадения внутри файлов (включая файлы из архивов) прямо на странице с подсветкой терминов, понимать по “светофору” где есть совпадения, видеть размер документа (char_count) и при необходимости быстро очистить поиск и вернуться к исходному состоянию.

### Критерии приёмки
1. Дерево файлов: при клике на элемент архива «report.zip» он разворачивается и показывает вложенные файлы/папки как обычная папка. Названия дочерних элементов соответствуют структуре архива. Неподдерживаемые файлы визуально «серые» и некликабельные.
2. Контекст заголовков: при выборе результата поиска из архива заголовок/хлебные крошки содержат имя архива и путь внутри, например: «report.zip › data/2024/q1/plan.docx».
3. Предпросмотр: в правой панели показывается сниппет(ы) текста с подсветкой всех терминов поиска; переход между совпадениями возможен стрелками/кнопками «след./пред.»; предпросмотр доступен для файлов, поддерживаемых индексом. Для неподдерживаемых — сообщение «Файл не поддерживается для предпросмотра».
4. Светофорные индикаторы: у каждого узла дерева отображается индикатор: зелёный — есть совпадения в этом узле (для файла) или в поддереве (для папки/архива); жёлтый — [NEEDS CLARIFICATION: требуется ли жёлтый статус как «есть совпадения в части потомков/не во всех» или «есть совпадения только в потомках, но не в текущем файле»?]; красный — совпадений нет.
5. char_count: под именем каждого файла показывается «Символов: N». Если значение неизвестно — «Символов: —». Для узла архива/папки агрегированное значение не показывается, чтобы не вводить в заблуждение.
6. Кнопка «Очистить все»: очищает строку поиска, снимает все подсветки в предпросмотре, сбрасывает индикаторы светофора в нейтральное состояние (серый/без индикатора), возвращает дерево в исходное раскрытие [NEEDS CLARIFICATION: сворачивать все узлы или сохранять текущее раскрытие?]. Файлы не удаляются.
7. Путь загрузки: после загрузки через выбор папки отображается под именем файла (или в боковой панели информации) исходный путь папки, из которой была произведена загрузка (относительно клиента), например: «Загружено из: Documents/Tenders/2024» [NEEDS CLARIFICATION: хранить и показывать путь клиента или только папку-ярлык? В целях приватности возможно показывать только последний сегмент].

### Краевые случаи
- В архиве присутствуют файлы неподдерживаемых форматов — они отображаются «серыми», предпросмотр недоступен, индикатор совпадений учитывает их только если поиск по индексу дал сниппеты (что не должно происходить, т.к. неподдерживаемые файлы не индексируются).
- В архиве повреждённый файл — в дереве есть элемент с пометкой об ошибке, предпросмотр недоступен, клики не приводят к падению UI; в логах сервера фиксируется ошибка извлечения на этапе индексации.
- Поиск выполнен без предварительно построенного индекса — система автоматически построит индекс и затем отобразит результаты (сообщение пользователю о прогрессе).
- Запрос пустой/слишком длинный/слишком много терминов — валидатор возвращает понятную ошибку, поиск не запускается.
- Большие архивы с глубокой вложенностью — дерево поддерживает ленивое раскрытие [NEEDS CLARIFICATION: требуется ли дозагрузка/пагинация узлов?].

## Требования

### Функциональные требования
- FR-001 (Навигация по архивам): Архивы ZIP/RAR в дереве файлов должны раскрываться и вести себя как обычные папки; вложенные элементы отображаются с исходными именами из архива. Неподдерживаемые файлы — некликабельны и «серые».
- FR-002 (Контекст заголовка): При выборе результата из архива интерфейс обязан показывать имя архива и путь внутри архива в виде хлебных крошек/заголовка результата.
- FR-003 (Предпросмотр со сниппетами): Для поддерживаемых файлов (включая внутри архивов) в правой панели отображаются сниппеты с подсветкой всех терминов; есть навигация между совпадениями и счётчик «k из n».
- FR-004 (Светофор совпадений): Отображать индикатор наличия совпадений у каждого узла дерева. Файл: зелёный при ≥1 совпадении, красный при 0. Папка/архив: зелёный, если в поддереве есть совпадения; красный, если в поддереве 0; жёлтый — [NEEDS CLARIFICATION: критерий жёлтого статуса].
- FR-005 (char_count под именем): Под именем каждого файла отображать «Символов: N» из индекса; при отсутствии данных — «Символов: —». Для папок/архивов агрегированное значение не выводить.
- FR-006 (Очистка поиска): Кнопка «Очистить все» очищает запрос, снимает подсветку и возвращает дерево и индикаторы в нейтральное состояние; результаты поиска и предпросмотр очищаются.
- FR-007 (Отображение пути загрузки): Для загруженных через выбор папки наборов — отображать путь исходной папки (человеко‑читаемо) рядом с файлами/в панели информации о файле.

### Ключевые сущности
- TreeNode (узел дерева): тип (папка/архив/файл/неподдерж.), имя, путь (виртуальный), флаги «кликабелен», индикатор совпадений.
- SearchResult (результат поиска): ссылка на узел, список сниппетов с подсветкой, количество совпадений.
- ArchiveNode (виртуальная папка-архив): имя архива, внутренний путь, дети.
- Preview (предпросмотр): текущий сниппет, позиция k из n, состояние подсветок.
- FileMeta (метаданные): char_count, поддерживаемость предпросмотра, источник загрузки (путь).
