# Feature Specification: Красная кнопка «Очистить все» рядом с «Выбрать папку»

**Feature Branch**: `002-`
**Created**: 2025-10-01
**Status**: Draft
**Input**: User description: "кнопка очистить все должна быть красного цвета и размещаться рядом с кнопкой выбрать папку. При нажатии на эту кнопку должны удаляться все папки и файлы, файл с индексом, результаты поиска, ключевые слова в строке поиска не должны быть удалены"

## User Scenarios & Testing (mandatory)

### Primary User Story
Как пользователь, я хочу видеть красную кнопку «Очистить все» рядом с кнопкой «Выбрать папку», чтобы одним действием удалить все загруженные папки/файлы, индекс и результаты поиска, при этом введённые ключевые слова в поле поиска сохранялись.

### Acceptance Scenarios
1. Given на главной странице есть кнопки «Выбрать папку» и «Очистить все», When я нажимаю «Очистить все», Then все загруженные папки/файлы удаляются из `uploads/`, индексный файл в `index/` удаляется, сохранённые результаты поиска очищаются, а строка поиска сохраняет введённые термины без изменений.
2. Given загружены файлы и построен индекс, When нажать «Очистить все», Then UI обновляется: дерево файлов пустое, индикаторы сброшены, индекса нет, но поле поиска остаётся с прежним текстом.

### Edge Cases
- Если папка `uploads/` пуста — кнопка всё равно доступна; нажатие приводит к безошибочному подтверждению и очистке состояния результатов/индекса.
- Если некоторые файлы заняты системой — операция не должна зависать; пользователю показывается краткое сообщение об ошибке для проблемных файлов, но процесс продолжается для остальных.
- Если индекс отсутствует — очистка индекса пропускается без ошибки.

## Requirements (mandatory)

### Functional Requirements
- FR-001: Система ДОЛЖНА отображать красную кнопку «Очистить все» рядом с кнопкой «Выбрать папку» на главной странице.
- FR-002: Нажатие «Очистить все» ДОЛЖНО удалять все файлы и папки из `uploads/` (только поддерживаемые и загруженные файлы пользователя), не затрагивая структуру приложения.
- FR-003: Нажатие «Очистить все» ДОЛЖНО удалять индексный файл `index/_search_index.txt` и связанные кеши результатов `index/search_results.json`.
- FR-004: Результаты поиска, статусы файлов и индикаторы ДОЛЖНЫ быть сброшены (в том числе в persistent-хранилище, если используется).
- FR-005: Строка поиска НЕ ДОЛЖНА очищаться — введённые пользователем ключевые слова сохраняются в поле и в состоянии UI.
- FR-006: Операция ДОЛЖНА выполняться быстро и не блокировать UI; ответ сервера — JSON с кратким итогом (кол-во удалённых файлов, наличие индекса до/после).
- FR-007: В случае ошибок удаления отдельных файлов система ДОЛЖНА продолжать очистку остальных и возвращать массив ошибок с относительными путями.
- FR-008: ДОЛЖНА быть защита от удаления вне `uploads/`/`index/` (валидация безопасного пути, без traversal).
- FR-009: ДОЛЖЕН быть эндпоинт, доступный из UI, для выполнения этой операции.

### Key Entities
- Кнопка «Очистить все»: элемент интерфейса, визуальный стиль — красный (danger), расположение — справа/слева от «Выбрать папку» (НЕОБХОДИМА УТОЧНЕНИЕ: сторона размещения относительно существующей вёрстки).
- Состояние поиска: строка поиска (значение сохраняется), результаты поиска (очищаются), индикаторы в дереве (сбрасываются).
